Авторизация по звонку

Авторизация по номеру телефона на данный момент самый распространенный способ аутентификации пользователей любых интернет-сервисов, мобильных приложений. Используется в качестве регистрации, подтверждения доступа, восстановления паролей и т.п.

На уровне законодательства, например, обязательная идентификации пользователей требуется в публичных сетях Wi-Fi в соответствии с Постановлениями Правительства РФ №758 и №801. Чтобы привязать mac-адрес устройства с которого выходят в сеть, необходимо подтвердить номер телефона.

Самыми популярными способами авторизации номера являются- авторизация по смс и по звонку.

Схема работы услуги авторизация по звонку:

1 вариант:

  • пользователь вносит свой телефонный номер в форму на сайте, интерфейсе приложения
  • данные через API передаются в наш сервис автоматического обзвона
  • система звонит на указанный номер, абонент берет трубку, голосовой робот произносит код\пароль
  • абонент вводит код\пароль в форму на сайте
  • поддерживает возможность подключения по smpp-протоколу

2 вариант:

  • данный вариант также называется -Flash Call
  • пользователь вносит свой телефонный номер в форму на сайте, интерфейсе приложения
  • данные через API передаются в наш сервис автоматического обзвона
  • система звонит на указанный номер, абонент берет трубку, вызов сбрасывается
  • абонент вводит последние 4-е цифры входящего номера телефона в форму на сайте

3 вариант:

  • пользователь вносит свой номер в форму на сайте и звонит на бесплатный многоканальный номер
  • после снятия трубки вызов сбрасывается
  • данные через API передаются в ваш сервис
  • вы сравнивает в своей БД указанный в форме номер с номером от которого был звонок

4 вариант:

  • пользователь вносит свой телефонный номер в форму на сайте(интерфейсе приложения)
  • данные через API передаются в наш сервис автоматического обзвона
  • система звонит на указанный номер, абонент берет трубку, голосовой робот в режиме диалога просит ответить на несколько вопросов
  • ответ распознаются нашей системой, данные через API передаются в ваш сервис и происходит авторизация

5 вариант:

  • пользователь вносит свой номер в форму на сайте и ему отображается код, который необходимо ввести позвонив на бесплатный многоканальный номер
  • после снятия трубки, система предлагает ввести код в тональном наборе
  • данные через API передаются в ваш сервис
  • вы сравнивает в своей БД указанный в форме номер от которого был звонок и код который ввели, и авторизуете пользователя

Авторизация по звонку происходит быстрее, чем авторизация через смс, т.к. смс могут приходить в телефон с задержками до нескольких минут. Также важным фактором является стоимость, цена отправки смс составляет в среднем 2,6 руб., а стоимость звонка- 0,35-0,65 руб. (в зависимости от кол-ва запросов в месяц).
