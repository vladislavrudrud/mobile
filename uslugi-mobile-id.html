Мобильный-ID – универсальное решение для безопасности в цифровом пространстве

Мобильный-ID —это межоператорский инструмент при помощи которого, используя только номер своего сотового телефона возможно заменить традиционный логин и пароль для входа в различные цифровые сервисы, а также совершения транзакций. Пользователь выбирает своего сотового оператора в качестве подтверждающей стороны своей личности и процесс входа становится автоматическим. Сервис работает по спецификациям Mobile Connect - мирового стандарта, разработанного Ассоциацией GSM (GSMA), в качестве основы он использует стандарт OpenID Connect с расширенными параметрами безопасности.

Регистрация

На основе Мобильного-ID возможна организация следующих услуг:

1. Аутентификация

позволяет зарегистрироваться или авторизоваться по номеру телефона при помощи Sim-push-уведомления. Используется при авторизации пользователей, вместо ввода логина/пароля, либо как первый шаг в двухфакторный авторизации. Подробнее о Sim-Push…

Схема работы:

  • Пользователь вводит свой номер телефона на вашем сайте/мобильном приложении
  • Вы по API отправляете запрос на оператора на аутентификацию номера
  • На экране телефона пользователя появляется Sim-push сообщение с запрос на подтверждение входа
  • После получения ответа пользователя оператор возвращает результат: Ок/Отмена
  • В случае положительного ответа происходит автоматический вход на сайт/мобильное приложение. Пользователю не нужно вводить логин/пароль

2. Seamless (Бесшовная аутентификация)

Сопоставление номера телефона, введенного пользователем для входа на веб-сайт/мобильное приложение с данные пользователя/sim-карты/сети/устройства у его оператора

Схема работы:

  • Пользователь вводит свой номер телефона для входа на ваш сайт/мобильное приложение
  • Вы по API отправляете запрос на проверку номера у обслуживающего оператора
  • Оператор сопоставляет данные пользователя/sim-карты/сети/устройства и возвращает ответ по результату проверки
  • В случае положительного ответа происходит автоматический вход на сайт/мобильное приложение. Пользователю не нужно вводить логин/пароль

3. Верификация данных

Проверка набора данных пользователя, которые оператор имеет возможность проверить на своей стороне, в соответствии с текущим законодательством. Используется, например, для подтверждения введенных пользователем данных в различных сервисах

Схема работы:

  • Вы по API отправляете запрос оператору на проверку данных о пользователе, оператор запрашивает у пользователя разрешение их проверить
  • Получив согласие пользователя, оператор возвращает ответ о результате проверки

4. Автозаполнение формы

Заполнение формы набором данных (Базовый, Стандартный, Расширенный) о пользователе (Идентификация пользователя). Используется при заполнении расширенного профиля в различных сервисах, оформления заявок на кредит, получение банковский карт и т.п.

Схема работы:

  • Пользователь выбирает на вашем сайте/мобильном приложении кнопку Мобильный-ID
  • Вы по API отправляете запрос оператору на предоставление данных, оператор запрашивает у пользователя разрешение их предоставить
  • Получив согласие пользователя, оператор возвращает ответ с набором данным
  • Вы заполняете полученными данными свою форму. Пользователю ничего не нужно самому вводить

ВАЖНО - при использовании услуг Мобильный ID телефон абонента должен быть подключен к мобильному интернету своего сотового оператора. Данные услуги ограниченны на сим-картах старого образца и устаревших моделях телефонов. – как-то выделить по аналогии с другими услугами
